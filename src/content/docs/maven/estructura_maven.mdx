En esta sección, exploraremos la estructura típica de un proyecto Maven y el archivo de configuración `pom.xml`, 
que es fundamental para la gestión de dependencias y la construcción del proyecto.

## Estructura típica de un proyecto Maven

<FileTree>

- mi_proyecto/
    - pom.xml
    - src/
        - main/
        - java/
    - test/

</FileTree>

## Ejemplo

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>es.vedruna</groupId>
    <artifactId>mi_proyecto</artifactId>
    <version>1.0-SNAPSHOT</version>
    <properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    </properties>
    <dependencies>
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter-api</artifactId>
            <version>5.7.1</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.8.1</version>
                <configuration>
                    <source>1.8</source>
                    <target>1.8</target>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>
```

### modelVersion

Especifica la versión del modelo de POM que se está utilizando. En este caso, se está utilizando la versión 4.0.0.

```xml
<modelVersion>4.0.0</modelVersion>
```
### groupId
Especifica el identificador único del grupo o la organización que desarrolla el proyecto. Generalmente, se utiliza el dominio invertido de la organización.

```xml
<groupId>es.vedruna</groupId>
```
### artifactId
Especifica el identificador único del proyecto o módulo. Este es el nombre del artefacto que se generará.

```xml
<artifactId>mi_proyecto</artifactId>
```
### version
Especifica la versión del proyecto. En este caso, se está utilizando la versión 1.0-SNAPSHOT, lo que indica que es una versión en desarrollo.

```xml
<version>1.0-SNAPSHOT</version>
``` 

### properties
Define propiedades del proyecto, como la versión de Java que se utilizará para compilar el código fuente. En este caso, se está utilizando Java 17.

```xml
<properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
</properties>
```

### dependencies
Define las dependencias del proyecto. En este caso, se está incluyendo la dependencia de JUnit Jupiter API para realizar pruebas unitarias.

```xml
<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-api</artifactId>
        <version>5.7.1</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```
### build
Define la configuración de construcción del proyecto. En este caso, se está utilizando el plugin de compilación de Maven para compilar el código fuente.

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.8.1</version>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
            </configuration>
        </plugin>
    </plugins>
</build>
```
